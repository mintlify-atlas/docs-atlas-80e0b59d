Get up and running with Serilog in just a few minutes. This guide will walk you through installing Serilog and creating your first structured log events.

## Prerequisites

- .NET 6.0 or later (or .NET Framework 4.6.2+)
- A .NET project (console app, ASP.NET Core, etc.)

## Installation

<Steps>
  <Step title="Install Serilog core package">
    Install the main Serilog package:
    
    <CodeGroup>
    ```bash .NET CLI
    dotnet add package Serilog
    ```
    
    ```bash Package Manager
    Install-Package Serilog
    ```
    
    ```xml PackageReference
    <PackageReference Include="Serilog" Version="4.*" />
    ```
    </CodeGroup>
  </Step>

  <Step title="Install sinks">
    Install one or more sinks to output your logs. For this quickstart, we'll use the Console and File sinks:
    
    <CodeGroup>
    ```bash .NET CLI
    dotnet add package Serilog.Sinks.Console
    dotnet add package Serilog.Sinks.File
    ```
    
    ```bash Package Manager
    Install-Package Serilog.Sinks.Console
    Install-Package Serilog.Sinks.File
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure the logger">
    Create and configure your logger in `Program.cs`:
    
    ```csharp
    using Serilog;

    Log.Logger = new LoggerConfiguration()
        .WriteTo.Console()
        .WriteTo.File("logs/app.log",
            rollingInterval: RollingInterval.Day,
            rollOnFileSizeLimit: true)
        .CreateLogger();

    try
    {
        Log.Information("Application starting up");
        
        // Your application code here
        
        Log.Information("Application shutting down");
    }
    catch (Exception ex)
    {
        Log.Fatal(ex, "Application terminated unexpectedly");
    }
    finally
    {
        await Log.CloseAndFlushAsync();
    }
    ```
  </Step>
</Steps>

## Your First Structured Log

The power of Serilog comes from structured logging with message templates:

```csharp
// Simple message
Log.Information("Hello, Serilog!");

// With properties (captured as structured data)
var name = "World";
Log.Information("Hello, {Name}!", name);

// With multiple properties
var user = "Alice";
var elapsed = 42;
Log.Information("User {User} completed operation in {Elapsed} ms", user, elapsed);

// Destructuring complex objects with @
var position = new { Latitude = 25.5, Longitude = -134.2 };
Log.Information("Received position {@Position}", position);
```

The `@` operator tells Serilog to serialize the object rather than calling `ToString()` on it.

## Understanding the API

### LoggerConfiguration

The `LoggerConfiguration` class is used to configure loggers. From `src/Serilog/LoggerConfiguration.cs:20`:

```csharp
public class LoggerConfiguration
{
    public LoggerSinkConfiguration WriteTo { get; }
    public LoggerMinimumLevelConfiguration MinimumLevel { get; }
    public LoggerEnrichmentConfiguration Enrich { get; }
    public LoggerFilterConfiguration Filter { get; }
    
    public Logger CreateLogger() { }
}
```

Key configuration options:
- **WriteTo**: Configure sinks (where logs go)
- **MinimumLevel**: Set the minimum log level
- **Enrich**: Add properties to all events
- **Filter**: Filter log events

### Log Levels

Serilog provides six log levels, defined in `src/Serilog/Events/LogEventLevel.cs`:

<Tabs>
  <Tab title="Information">
    Use for general informational messages about application flow:
    
    ```csharp
    Log.Information("User {UserId} logged in", userId);
    ```
  </Tab>
  
  <Tab title="Warning">
    Use for unexpected but recoverable situations:
    
    ```csharp
    Log.Warning("Disk space low: {FreeSpace} GB remaining", freeSpace);
    ```
  </Tab>
  
  <Tab title="Error">
    Use for errors and exceptions:
    
    ```csharp
    try
    {
        ProcessOrder(order);
    }
    catch (Exception ex)
    {
        Log.Error(ex, "Failed to process order {OrderId}", order.Id);
    }
    ```
  </Tab>
  
  <Tab title="Debug">
    Use for diagnostic information:
    
    ```csharp
    Log.Debug("Loading configuration from {Path}", configPath);
    ```
  </Tab>
</Tabs>

## Message Template Patterns

### Capturing Properties

```csharp
// Named properties (recommended)
Log.Information("User {UserId} from {Country}", "alice", "US");

// Positional properties
Log.Information("User {0} from {1}", "alice", "US");
```

### Destructuring Operators

```csharp
// @ destructures objects (serializes structure)
var user = new User { Id = 1, Name = "Alice" };
Log.Information("Created user {@User}", user);
// Output: Created user {"Id": 1, "Name": "Alice"}

// $ stringifies (calls ToString)
Log.Information("Created user {$User}", user);
// Output: Created user "User: Alice"

// No operator (captures scalar value)
Log.Information("Created user {User}", user);
// Output: Created user User: Alice
```

## Using ILogger Interface

For dependency injection scenarios, use the `ILogger` interface from `src/Serilog/ILogger.cs`:

```csharp
public class OrderService
{
    private readonly ILogger _logger;
    
    public OrderService(ILogger logger)
    {
        _logger = logger.ForContext<OrderService>();
    }
    
    public void ProcessOrder(Order order)
    {
        _logger.Information("Processing order {OrderId}", order.Id);
        
        // ... process order ...
        
        _logger.Information("Order {OrderId} completed in {Elapsed} ms", 
            order.Id, elapsed);
    }
}
```

<Note>
The `ForContext<T>()` method enriches all log events from this logger with the source context (the type name).
</Note>

## ASP.NET Core Integration

For ASP.NET Core applications, configure Serilog in `Program.cs`:

```csharp
using Serilog;

var builder = WebApplication.CreateBuilder(args);

// Configure Serilog
builder.Host.UseSerilog((context, services, configuration) => configuration
    .ReadFrom.Configuration(context.Configuration)
    .ReadFrom.Services(services)
    .Enrich.FromLogContext()
    .WriteTo.Console()
    .WriteTo.File("logs/app.log", rollingInterval: RollingInterval.Day));

var app = builder.Build();

// Use Serilog request logging
app.UseSerilogRequestLogging();

app.Run();
```

Then configure log levels in `appsettings.json`:

```json
{
  "Serilog": {
    "MinimumLevel": {
      "Default": "Information",
      "Override": {
        "Microsoft": "Warning",
        "System": "Warning"
      }
    }
  }
}
```

## Common Patterns

### Contextual Logging

Add properties to all events within a scope:

```csharp
using Serilog.Context;

using (LogContext.PushProperty("CorrelationId", correlationId))
{
    // All log events in this scope will include CorrelationId
    Log.Information("Processing request");
}
```

### Type-Specific Loggers

Create loggers specific to a type for better filtering:

```csharp
var logger = Log.ForContext<MyService>();
logger.Information("Service started");
// Includes SourceContext property: "MyService"
```

### Conditional Logging

Check if a level is enabled before expensive operations:

```csharp
if (Log.IsEnabled(LogEventLevel.Debug))
{
    var diagnostics = GatherExpensiveDiagnostics();
    Log.Debug("Diagnostics: {@Diagnostics}", diagnostics);
}
```
